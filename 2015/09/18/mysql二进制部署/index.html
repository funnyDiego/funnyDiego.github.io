mysql 二进制部署

 2015-09-18 16:11:27

 2.1k Words  11 Mins  

学习 hadoop 主要参考 hadoop官网，本文主要介绍伪分布式模式的搭建（单节点）及注意事项。

本文介绍 MySQL 的二进制部署，参考资料：https://github.com/Hackeruncle/mysql。

MySQL 版本：5.6.23

1.上传下载好的tar包

rz mysql-5.6.23-linux-glibc2.5-x86_64.tar.gz

2.移动到 /usr/local 目录下

mv ./mysql-5.6.23-linux-glibc2.5-x86_64.tar.gz /usr/local/

3.查看是否已经安装MySQL（比如：系统自带）

4.tar包解压，创建软连接

5.创建 user 和 group

6.创建 MySQL 配置文件

vi /etc/my.cnf

进入编辑模式，粘贴下述类容（生产环境调优时，修改 innodb_buffer_pool_size = 2048M 即可，一般6G、8G、12G 就足以）

保存，退出，该配置文件已创建成功

7.改变配置文件权限，第一次安装

新建放置 binlog 文件夹 arch
[mysqladmin@bigdata ~]$ mkdir arch

执行命令 scripts/mysql_install_db

退到 root 用户安装 libaio
[root@bigdata local]# yum -y install libaio

再次安装

8.配置 mysql 服务，设置开机自启动

root 用户下：

将服务文件拷贝到init.d下，并重命名为mysql

[root@bigdata mysql]# cp support-files/mysql.server /etc/rc.d/init.d/mysql

赋予可执行权限

[root@bigdata mysql]# chmod +x /etc/rc.d/init.d/mysql
[root@bigdata mysql]# ll /etc/rc.d/init.d/mysql
-rwxr-xr-x 1 root root 10880 Dec 5 23:27 /etc/rc.d/init.d/mysql

如果之前部署过 mysql 服务，可以删除服务

[root@bigdata mysql]# chkconfig --del mysql

第一次安装时，添加服务

[root@bigdata mysql]# chkconfig --add mysql
[root@bigdata mysql]# chkconfig --level 345 mysql on

[root@bigdata mysql]# vi /etc/rc.local

进入编辑模式，新增内容：

su - mysqladmin -c "/etc/init.d/mysql start --federated"

9.启动 mysql 服务，查看服务和监听状态

切换用户 mysqladmin

启动 mysql 服务和查看 mysql 服务状态

登录 mysql

查看 mysql 进程，获取 PID：1022889

查看端口号：3306

``[root@bigdata local]# netstat -ntlp | grep 1022830 tcp6 0 0 :::3306 :::* LISTEN 1022830/mysqld

10.登录 mysql

[mysqladmin@bigdata ~]$ mysql

有密码的时候

11.修改密码和清理用户

12.修改配置文件 .bashrc

切换到 mysqladmin 用户，编辑配置文件 ~/.bashrc
[mysqladmin@bigdata ~]$ vi .bashrc
新增环境变量配置
export MYSQL_HOME=/usr/local/mysql
export PATH=$MYSQL_HOME/bin:$PATH

[mysqladmin@bigdata ~]$ source .bashrc
[mysqladmin@bigdata ~]$ echo $MYSQL_HOME
/usr/local/mysql

附加

修改配置文件 ~/.bashrc

[mysqladmin@bigdata ~]$ vi .bashrc

新增如下配置
PS1=uname -n":"'$USER'":"'$PWD'":>"; export PS1

效果如下：

可直观地查看当前所在的目录。

重新部署

如果部署出了问题
1）检查执行的目录、用户
2）重新部署
i) rm -rf arch/* binlog 文件，为了保持有效期，会有自动删除机制
ii) rm -rf data/* 数据
iii) 再次从执行下述命令开始：scripts/mysql_install_db –user=mysqladmin –basedir=/usr/local/mysql –datadir=/usr/local/mysql/data
